# Хуердли — визуальный стиль

## Продукт

- Русская ежедневная word-игра (5–6 букв, 6 попыток), аналог Wordle с характером.
- Аудитория: русскоязычные, 18+ в матовые дни (Пн/Ср/Пт по Москве).
- Стек UI: vanilla HTML (`index.html`), CSS (`src/style.css`), рендер в `src/main.ts`. Без React, без Tailwind.

## Фирменные элементы (сохранять)

- Тёмная тема, но **не клон Wordle 1:1** — свой характер.
- Пустые клетки: SVG-«хуй» на фоне (`.cell.empty::before`) — фирменная деталь, можно усиливать, не убирать без запроса.
- Цвета состояний: correct (зелёный), present (жёлтый), absent (серый) — узнаваемая механика Wordle, оттенки можно менять.
- Анимации: flip при открытии букв, shake при ошибке — обязательны; уважать `prefers-reduced-motion`.

## Ограничения

- Минимальный diff: не трогать игровую логику без необходимости.
- CSS-переменные в `:root` — предпочтительный способ менять палитру и размеры.
- Мобилка first: брейкпоинты 420px и 360px уже есть — расширять, не ломать.
- Touch targets клавиш ≥ 44px на маленьких экранах.
- Контраст текста ≥ 4.5:1.
- `VITE_BASE_PATH` для GitHub Pages — пути к ассетам через Vite base, не хардкодить `/`.

## Тон копирайта в UI

- Коротко, по-русски, с юмором где уместно (победа/поражение).
- Ошибки — конкретные («Нет в словаре»), не абстрактные.

## Процесс редизайна

1. Сначала токены: палитра, типографика, размеры клеток/клавиш.
2. Потом компоненты: board → keyboard → header → modals.
3. Проверить на узкой ширине (360px) и с виртуальной клавиатурой (safe-area).
4. Локально: `npm run dev` или `npm run preview:pages`.
