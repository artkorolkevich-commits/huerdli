### 9. ~~Ротация слов без повторов~~ ✅

| | |
|---|---|
| **ID** | `word-no-repeat` |
| **Тип** | Улучшение |
| **Приоритет** | P1 |
| **Статус** | done |
| **Файлы** | `src/lib/daily.ts`, тесты/скрипт проверки расписания |

**Проблема:** сейчас слово дня — случайный индекс в словаре через `mulberry32(hash(dateKey))` (`pickDailyWord`). Повторы возможны в любой день; порядок файлов словаря — алфавитный, но на выдачу это не влияет напрямую, зато при смене алгоритма нельзя просто брать слова по порядку из `.txt`.

**Нужно:**
- Пока **не отыграны все слова** из соответствующего пула — **без повторов**
- Порядок выдачи — **не алфавитный** (детерминированное перемешивание, а не `words[i]` по файлу)

**Четыре независимых пула** (как сейчас в `pickDailyWord`):
- обычные 5 букв (~3370 слов)
- обычные 6 букв (~4715)
- мат 5 букв (~85)
- мат 6 букв (~76)

День выбирает длину (`pickWordLength`) и мат/не мат (`isProfanityDay`) → один из пулов. Счётчик «какое по счёту слово из этого пула» ведётся только по дням, которые реально использовали этот пул (с `GAME_EPOCH`).

**Черновой алгоритм:**
1. При старте (или на этапе сборки) для каждого пула — **фиксированный shuffle** (seed + FNV/mulberry32, как сейчас для детерминизма у всех игроков)
2. Для `dateKey`: определить пул → посчитать `slot = число прошедших дней с epoch, где этот же пул` → `word = shuffled[slot % pool.length]`
3. После полного круга (`slot >= pool.length`): новый shuffle с seed `v1+cycle` или явная политика повтора (зафиксировать)

**Ограничения:**
- Один и тот же `dateKey` → одно слово у всех (сохранить)
- `DAILY_WORD_OVERRIDES` — по-прежнему работают
- Смена алгоритма сдвинет прошлые слова дня — нужна версия схемы (`word-schedule-v2`) и осознанный cutover

**Вопросы для проработки:**
- Что делать после исчерпания пула: новый цикл shuffle или стоп?
- Нужен ли CLI `npm run word-schedule -- --from 2026-06-03 --days 30` для превью расписания?
- Учитывать ли смену размера словаря при деплое (пересчёт цикла)?

**Критерии готовности:**
- [ ] В пределах одного цикла пула слово не повторяется
- [ ] Порядок не совпадает с алфавитом файла словаря
- [ ] Детерминизм: тот же день → то же слово на всех устройствах
- [ ] Матовые и обычные дни ротируют свои пулы независимо
- [ ] Документировано в README (как считается слово дня)
